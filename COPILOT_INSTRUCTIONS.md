# Copilot Instructions

## Purpose
This file provides guidelines for using GitHub Copilot to assist in the development of the "Workitems Manager" extension. It also includes instructions for maintaining the `DOCUMENTATION.md` file to ensure it stays up-to-date with the latest features and changes.

---

## Guidelines for Using Copilot

1. **Code Suggestions**:
   - Use Copilot to generate boilerplate code, repetitive patterns, and utility functions.
   - Review all suggestions carefully to ensure they align with the project's requirements and coding standards.

2. **Feature Implementation**:
   - When implementing new features, ensure that the logic integrates seamlessly with the existing codebase.
   - Use Copilot to assist with repetitive tasks, such as registering commands or updating data structures.

3. **Testing**:
   - Use Copilot to generate test cases for new features.
   - Ensure all tests are comprehensive and cover edge cases.

4. **Error Handling**:
   - Use Copilot to suggest error-handling patterns.
   - Ensure all error messages are user-friendly and informative.

---

## Maintaining `DOCUMENTATION.md`

### Instructions
- **Mandatory Updates**:
  - Every time a new feature is implemented or an existing feature is modified, update the `DOCUMENTATION.md` file.
  - Include details such as:
    - Feature description.
    - Commands added or modified.
    - Changes to the tree view or data structure.

- **Structure**:
  - Follow the existing structure of the `DOCUMENTATION.md` file.
  - Use clear and concise language.

- **Examples**:
  - Provide examples or screenshots where applicable.

### Automation
- Use the following steps to ensure `DOCUMENTATION.md` is updated:
  1. After implementing a feature, open the `DOCUMENTATION.md` file.
  2. Use Copilot to suggest updates based on the changes made.
  3. Review and refine the suggestions to ensure accuracy.

---

## Best Practices
- **Consistency**:
  - Maintain consistent formatting and style across all documentation.
- **Clarity**:
  - Ensure all instructions and descriptions are easy to understand.
- **Versioning**:
  - Include version numbers or dates for significant updates.

---

By following these instructions, you can ensure that the "Workitems Manager" extension remains well-documented and easy to maintain.
